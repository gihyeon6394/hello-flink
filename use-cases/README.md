# Use Cases

Apache Flink는 **스트림·배치 처리 통합 지원**, **정교한 상태 관리**, **이벤트 시간(event-time) 처리**, **Exactly-once 일관성 보장**을 제공하는 강력한 스트림 처리
프레임워크다.  
YARN, Kubernetes, Standalone 환경에서 실행 가능하며, 고가용성 구성 시 단일 장애 지점이 없다. 수천 코어와 수 TB 상태까지 확장 가능하고, **고처리량·저지연** 특성으로 대규모 실시간
애플리케이션에서 검증되었다.

Flink 기반 애플리케이션은 크게 다음 세 가지로 나뉜다.

- Event-driven Applications
- Data Analytics Applications
- Data Pipeline Applications

### Event-driven Applications

이벤트 기반 애플리케이션은 하나 이상의 이벤트 스트림을 입력으로 받아, 이벤트 도착 시 **계산·상태 갱신·외부 액션**을 수행하는 **상태 기반 애플리케이션**이다.

기존의 계층형 아키텍처(애플리케이션 + 원격 DB)와 달리,  
이벤트 기반 애플리케이션은 **상태와 연산이 함께 위치**하여 로컬(메모리/디스크) 접근이 가능하다. 장애 복구는 원격 스토리지로의 주기적 체크포인트로 해결한다.

![img.png](img.png)

#### 장점

- 로컬 상태 접근으로 **높은 처리량과 낮은 지연시간**
- 비동기·점진적 체크포인팅으로 성능 영향 최소화
- 애플리케이션별 상태 관리 → 스키마 변경·스케일 조정 시 **조정 비용 감소**

#### Flink의 지원

- 대규모 상태(수 TB) 관리 + Exactly-once 보장
- Event-time, 커스텀 윈도우, ProcessFunction 기반 세밀한 시간 제어
- CEP 라이브러리를 통한 이벤트 패턴 탐지
- **Savepoint**를 통한 애플리케이션 업데이트, 스케일 변경, A/B 테스트

#### 대표 사례

- 사기 탐지
- 이상 탐지
- 룰 기반 알림
- 비즈니스 프로세스 모니터링
- 웹/소셜 애플리케이션

### Data Analytics Applications

데이터 분석 애플리케이션은 원시 데이터에서 **인사이트를 추출**한다.  
전통적으로는 배치 기반 분석이 일반적이었으나, 스트림 처리 엔진을 사용하면 **실시간 분석**이 가능하다.

스트리밍 분석은 이벤트를 지속적으로 소비하며 결과를 **계속 갱신**하고, 결과는 외부 DB에 저장하거나 애플리케이션 내부 상태로 유지된다.

![img_1.png](img_1.png)

#### 장점

- 배치 대비 **훨씬 낮은 지연시간**
- 주기적 배치 경계로 인한 인위적 데이터 단절 문제 제거
- 단일 애플리케이션 내에서 ingestion → 계산까지 처리 → **아키텍처 단순화**
- 엔진 자체의 장애 복구 메커니즘 활용 가능

#### Flink의 지원

- 스트림·배치 통합 SQL (ANSI SQL 지원)
- 동일 쿼리를 배치/스트림에서 동일한 의미로 실행
- UDF 지원
- 복잡한 로직은 DataStream / DataSet API로 확장 가능

#### 대표 사례

- 통신 네트워크 품질 모니터링
- 모바일 앱 실험·제품 업데이트 분석
- 실시간 사용자 데이터 분석
- 대규모 그래프 분석

### Data Pipeline Applications

데이터 파이프라인은 **ETL과 유사**하지만, 주기적 실행이 아닌 **지속적 스트리밍 방식**으로 동작한다.  
지속적으로 생성되는 데이터를 읽어 **낮은 지연시간으로 변환·전달**한다.

예:

- 파일 시스템 디렉터리 감시 → 이벤트 로그로 적재
- 이벤트 스트림을 DB에 머티리얼라이즈
- 검색 인덱스를 점진적으로 구축

![img_2.png](img_2.png)

#### 장점

- 주기적 ETL 대비 **데이터 전달 지연 최소화**
- 지속적 입력·출력이 가능해 활용 범위가 넓음

#### Flink의 지원

- SQL / Table API 기반 데이터 변환·보강
- 복잡한 요구사항은 DataStream API로 처리
- Kafka, Kinesis, Elasticsearch, JDBC 등 다양한 커넥터 제공
- 파일 시스템 연속 소스 및 시간 단위 파일 싱크 지원
